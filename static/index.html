Docker — Common Commands

List running containers
docker ps
List all containers (including stopped)
docker ps -a
View container logs (last 50 lines)
docker logs <container> --tail 50
Follow container logs
docker logs -f <container>
Start / stop / restart container
docker start|stop|restart <container>
Shell into a running container
docker exec -it <container> /bin/sh
Remove stopped containers
docker container prune
Remove unused images
docker image prune -a
Inspect container
docker inspect <container>
Show containers on a network
docker network inspect <network> --format '{{range .Containers}}{{.Name}} {{end}}'
Connect container to network
docker network connect <network> <container>

Docker Compose — Common Commands

Start services (detached)
docker compose up -d
Start and rebuild images
docker compose up -d --build
Stop services
docker compose down
Stop and remove volumes
docker compose down -v
View logs for all services
docker compose logs -f
Pull latest images
docker compose pull

Linux — Common Commands

Disk usage (human readable)
df -h
Directory size
du -sh <dir>
Memory usage
free -h
Running processes (top 10 by CPU)
ps aux --sort=-%cpu | head -10
Check open ports
ss -tlnp
Tail a log file
tail -f /var/log/syslog
Search running processes
ps aux | grep <name>
Check systemd service status
systemctl status <service>
Reload systemd after editing unit files
systemctl daemon-reload

Deploying a New Project

1. Create project directory
mkdir -p /home/runner/<project> && cd /home/runner/<project>
2. Create .env file
nano .env
3. Create docker-compose.yml — attach services to proxy network
services: app: image: ghcr.io/<owner>/<image>:${APP_VERSION} restart: unless-stopped expose: - "80" networks: - default - proxy networks: default: {} proxy: external: true
4. Add to Caddyfile
<subdomain>.thekiffy.co.uk { reverse_proxy <service-name>:80 }
5. Deploy
docker compose up -d
6. Reload Caddy
docker exec caddy-caddy-1 caddy reload --config /etc/caddy/Caddyfile

Setting Up a GitHub Actions Runner

1. Create runner directory
mkdir -p /home/runner/actions-runner-<project> && cd /home/runner/actions-runner-<project>
2. Download runner (check GitHub for latest version)
curl -o runner.tar.gz -L https://github.com/actions/runner/releases/download/v2.322.0/actions-runner-linux-x64-2.322.0.tar.gz tar xzf runner.tar.gz
3. Register with your repo (get token from repo → Settings → Actions → Runners)
./config.sh --url https://github.com/<owner>/<repo> --token <token>
4. Install and start as a service
sudo ./svc.sh install sudo ./svc.sh start
5. Check status
sudo ./svc.sh status
